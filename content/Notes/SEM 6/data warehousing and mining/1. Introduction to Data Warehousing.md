# 1.1 Data Warehousing

## 1. Definition of Data Warehousing

Data Warehousing is the process of collecting, storing, integrating, and managing large amounts of data from different sources in a centralized repository called a Data Warehouse for the purpose of analysis, reporting, and decision-making.

A Data Warehouse stores historical and current data in an organized form so that organizations can analyze trends, generate reports, and support business intelligence activities.

The concept of Data Warehousing was introduced by Bill Inmon.

### Standard Definition by Bill Inmon

A Data Warehouse is:

> “Subject-oriented, Integrated, Time-variant, and Non-volatile collection of data used to support management decision making.”

---

## 2. Characteristics of Data Warehouse

### 1. Subject-Oriented

A Data Warehouse is organized around major business subjects instead of business processes.

Examples of subjects:

- Sales
    
- Customers
    
- Products
    
- Finance
    
- Employees
    

Operational databases focus on daily operations, while data warehouses focus on analysis of subjects.

Example:

A retail company may analyze:

- Product sales
    
- Customer buying patterns
    
- Monthly profit
    

instead of handling individual transactions.

---

### 2. Integrated

Data is collected from multiple heterogeneous sources and integrated into a consistent format.

Different systems may use:

- Different naming conventions
    
- Different file formats
    
- Different units
    

The warehouse standardizes all data.

Example:

|Source|Customer Gender Format|
|---|---|
|System A|M/F|
|System B|Male/Female|

In Data Warehouse:

- All values may be converted to “Male” and “Female”.
    

This improves consistency.

---

### 3. Time-Variant

Data Warehouse stores historical data over long periods.

Each record is associated with a time dimension.

Operational systems mostly store current data, while warehouses maintain years of historical data.

Example:

A company may store:

- Sales data of last 10 years
    
- Employee performance over several years
    
- Monthly revenue trends
    

This helps in trend analysis and forecasting.

---

### 4. Non-Volatile

Once data enters the warehouse, it is not frequently updated or deleted.

Operations mainly performed:

- Data Loading
    
- Data Retrieval
    
- Data Analysis
    

Unlike operational systems, transactions such as insert, update, and delete are minimal.

Example:

Past year sales records remain unchanged for future analysis.

---

## 3. Purpose of Data Warehouse

### 1. Decision Support

Helps managers and executives make strategic decisions using analytical reports.

Example:

- Identifying profitable products
    
- Understanding customer behavior
    

---

### 2. Data Analysis

Allows organizations to analyze huge volumes of data efficiently.

Example:

- Year-wise sales analysis
    
- Regional performance comparison
    

---

### 3. Historical Record Maintenance

Stores long-term historical data.

Example:

- Sales trends over 15 years
    
- Market growth patterns
    

---

### 4. Business Intelligence

Supports:

- Reporting
    
- Dashboards
    
- Data mining
    
- Forecasting
    

---

### 5. Improved Data Quality

Data from different sources is cleaned and standardized.

This removes:

- Duplicate data
    
- Inconsistencies
    
- Errors
    

---

# 1.2 Difference Between Operational Database System and Data Warehouse

## 1. Operational Database System

Operational Database Systems are databases used to manage day-to-day business operations and transactions.

These systems are also called:

- OLTP Systems
    
- Online Transaction Processing Systems
    

### Features

- Handles real-time transactions
    
- Supports insert, update, delete operations
    
- Stores current data
    
- Optimized for fast transaction processing
    

### Examples

- ATM systems
    
- Railway reservation systems
    
- Banking applications
    
- E-commerce order systems
    

---

## 2. Data Warehouse

A Data Warehouse is used for analysis and decision support rather than daily transactions.

These systems are called:

- OLAP Systems
    
- Online Analytical Processing Systems
    

### Features

- Stores historical data
    
- Used for reporting and analysis
    
- Optimized for complex queries
    
- Supports managerial decisions
    

### Examples

- Sales trend analysis
    
- Business intelligence dashboards
    
- Market forecasting systems
    

---

## 3. Comparison between OLTP and OLAP Systems

|Basis|OLTP|OLAP|
|---|---|---|
|Full Form|Online Transaction Processing|Online Analytical Processing|
|Purpose|Daily operations|Data analysis|
|Users|Clerks, operators|Managers, analysts|
|Data Type|Current data|Historical data|
|Query Type|Simple queries|Complex analytical queries|
|Operations|Insert, update, delete|Read and analyze|
|Response Time|Milliseconds|Seconds or minutes|
|Database Design|Highly normalized|Denormalized|
|Number of Records|Current transactions|Large historical data|
|Example|Banking system|Business reporting system|

---

# 1.5 Metadata

## 1. Definition of Metadata

Metadata means “data about data”.

It provides information describing:

- Structure of data
    
- Source of data
    
- Meaning of data
    
- Relationships among data
    

Metadata helps users understand and manage warehouse data effectively.

Example:

For a sales column:

- Data type
    
- Source table
    
- Last update date
    
- Description
    

all form metadata.

---

## 2. Role of Metadata in Data Warehouse

Metadata acts as a directory or roadmap of the Data Warehouse.

### Functions

- Describes warehouse structure
    
- Helps locate data
    
- Supports ETL process
    
- Assists query optimization
    
- Maintains data consistency
    

---

## 3. Classification of Metadata

### 1. Technical Metadata

Contains technical information about data.

 Includes

- Table names
    
- Column names
    
- Data types
    
- Indexes
    
- Schema definitions
    

### Example

Information about database tables and relationships.

---

### 2. Business Metadata

Contains business-related definitions understandable by business users.

 Includes

- Business rules
    
- Definitions
    
- Calculations
    
- KPIs
    

### Example

Definition of “Net Profit”.

---

### 3. Operational Metadata

Contains information about warehouse operations.

### Includes

- Data loading time
    
- Backup details
    
- Refresh schedules
    
- Error logs
    

### Example

Last ETL execution status.

---

## 4. Importance of Metadata

### 1. Data Understanding

Helps users understand meaning and structure of data.

---

### 2. Data Management

Assists administrators in managing warehouse efficiently.

---

### 3. Query Optimization

Helps database systems execute queries efficiently.

---

### 4. Data Integration Support

Supports combining data from multiple sources consistently.

---

# 1.6 Data Marts

## 1. Definition of Data Mart

A Data Mart is a smaller and specialized subset of a Data Warehouse designed for a specific department or business function.

It contains focused data relevant to a particular group of users.

Example:

- Sales Data Mart
    
- HR Data Mart
    
- Finance Data Mart
    

---

## 2. Types of Data Marts

### 1. Dependent Data Mart

Created from an existing Data Warehouse.

Data is extracted from centralized warehouse.

---

### 2. Independent Data Mart

Created directly from operational systems without a central warehouse.

---

### 3. Hybrid Data Mart

Combines data from:

- Data Warehouse
    
- Operational systems
    
- External sources
    

---

## 3. Reasons for Creating Data Marts

### 1. Department-Specific Analysis

Provides customized data for departments.

Example:  
HR department accesses employee-related data only.

---

### 2. Faster Access to Data

Smaller size improves query speed.

---

### 3. Reduced Implementation Cost

Cheaper than implementing a full enterprise warehouse.

---

### 4. Improved Performance

Less data volume improves system efficiency.

---

## 4. Building Data Marts

### Top-Down Approach

#### 1. Enterprise Data Warehouse First

A central warehouse is created first.

---

### 2. Data Marts Created from Warehouse

Department-specific marts are later generated.

### Advantages

- High consistency
    
- Better integration
    

### Disadvantages

- Expensive
    
- Time-consuming
    

---

# Bottom-Up Approach

## 1. Data Marts Created First

Individual departments build their own marts.

---

## 2. Combined Later into Warehouse

Data marts are integrated later.

### Advantages

- Faster implementation
    
- Lower initial cost
    

### Disadvantages

- Data inconsistency risk
    
- Integration difficulty
    

---

# 1.7 Data Warehouse Architecture

## 1. Two-Tier Architecture

A two-tier architecture directly connects clients with the warehouse server.

Components

### 1. Client Layer

Used by end users for:

- Querying
    
- Reporting
    
- Analysis
    

### 2. Data Warehouse Server Layer

Stores warehouse database and handles queries.

### Advantages

- Simple architecture
    
- Faster communication
    

### Disadvantages

- Limited scalability
    
- Heavy server load
    

---

# 2. Three-Tier Architecture

Most commonly used architecture.

---

## 1. Bottom Tier

Contains:

- Data sources
    
- ETL tools
    
- Data Warehouse database
    

### Functions

- Data extraction
    
- Cleaning
    
- Loading
    

---

## 2. Middle Tier

Contains OLAP server.

### Functions

- Query processing
    
- Data aggregation
    
- Multidimensional analysis
    

---

## 3. Top Tier

Contains front-end tools for users.

### Examples

- Dashboards
    
- Reporting tools
    
- Data mining applications
    

---

# 1.8 Data Warehouse Schema

# 1. Star Schema

A Star Schema consists of:

- One central fact table
    
- Multiple dimension tables
    

It resembles a star shape.

---

## 1. Fact Table

Contains:

- Numerical measures
    
- Foreign keys
    

Example:  
Sales amount, quantity sold.

---

## 2. Dimension Tables

Contain descriptive information.

Example:

- Product details
    
- Customer information
    
- Time data
    

---

## 3. Characteristics and Advantages

### Characteristics

- Denormalized structure
    
- Simple design
    

### Advantages

- Faster query performance
    
- Easy understanding
    
- Efficient reporting
    

---

# 2. Snowflake Schema

A Snowflake Schema is an extension of Star Schema where dimension tables are normalized.

---

## 1. Normalized Dimension Tables

Dimension tables are split into multiple related tables.

Example:  
Location dimension divided into:

- City
    
- State
    
- Country
    

---

## 2. Structure and Features

### Features

- Reduced redundancy
    
- More complex joins
    
- Better storage efficiency
    

### Disadvantages

- Slower queries
    
- Complex structure
    

---

# 3. Fact Constellation Schema

Also called Galaxy Schema.

Contains:

- Multiple fact tables
    
- Shared dimension tables
    

---

## 1. Multiple Fact Tables

Example:

- Sales fact table
    
- Shipping fact table
    

---

## 2. Shared Dimension Tables

Same dimensions are shared among fact tables.

Example:

- Time dimension
    
- Product dimension
    

---

# 1.9 OLAP (Online Analytical Processing)

# 1. Need for OLAP

OLAP systems allow fast multidimensional analysis of warehouse data.

---

## 1. Multidimensional Analysis

Data can be viewed from multiple perspectives.

Example:  
Sales analyzed by:

- Time
    
- Region
    
- Product
    

---

## 2. Decision Support

Supports strategic business decisions.

Example:  
Identifying profitable markets.

---

## 3. Fast Analytical Queries

Optimized for complex calculations and aggregation.

---

# 2. OLAP Operations

## 1. Roll-Up

Aggregates data from lower level to higher level.

Example:  
City sales → State sales → Country sales

---

## 2. Drill-Down

Moves from summarized data to detailed data.

Example:  
Yearly sales → Monthly sales → Daily sales

---

## 3. Slice

Selects a single dimension value.

Example:  
Sales data for year 2025 only.

---

## 4. Dice

Selects multiple dimensions and values.

Example:  
Sales for:

- Product A
    
- Region West
    
- Year 2025
    

---

## 5. Pivot

Rotates data axes for different views.

Example:  
Rows converted to columns in reports.

---

# 3. OLAP Models

# 1. ROLAP (Relational OLAP)

Stores data in relational databases.

### Features

- Scalable
    
- Handles large data volumes
    

### Disadvantages

- Slower query performance
    

---

# 2. MOLAP (Multidimensional OLAP)

Stores data in multidimensional cubes.

### Features

- Very fast query response
    
- Precomputed aggregations
    

### Disadvantages

- Higher storage requirement
    

---

# 3. HOLAP (Hybrid OLAP)

Combination of ROLAP and MOLAP.

### Features

- Balance between speed and scalability
    
- Stores summary data in cubes and detailed data in relational tables
    

### Advantages

- Improved performance
    
- Efficient storage usage
