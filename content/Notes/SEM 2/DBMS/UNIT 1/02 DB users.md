### 1. Database Administrators (DBA)
1. the <mark style="background: #2dba4e;">most important</mark> type of database users in DBMS.
2. Database Administrator is <mark style="background: #FFF3A3A6;">an individual or a team of users</mark>
3. who <mark style="background: #2dba4e;">define the database schema</mark> 
4. <mark style="background: #FFB8EBA6;">takes charge of controlling various levels of the database</mark> within the organization.
5. have <mark style="background: #FFB86CA6;">full control</mark> of the database
6. known as the<mark style="background: #FFB86CA6;"> super-users </mark>of the database.
7. They <mark style="background: #FFF3A3A6;">work alongside developers</mark> in order to <mark style="background: #2dba4e;">discuss and design the overall structure of the database</mark> 
8. design structures including <mark style="background: #FF5582A6;">layouts</mark>, <mark style="background: #FFF3A3A6;">functionalities</mark>, <mark style="background: #2dba4e;">workflow</mark>, etc.
9. <mark style="background: #2dba4e;">DBAs have to provide login credentials to all</mark>
10. responsible for <mark style="background: #FFB86CA6;">providing security</mark> to the database by <mark style="background: #FF5582A6;">restricting unauthorized users</mark> from accessing the database.
11. DBAs have <mark style="background: #FFF3A3A6;">all the privileges</mark> allowed by the DBMS.
12. have to <mark style="background: #FFB8EBA6;">update the database timely</mark> in terms of 
	1. technology
	2. functionality
	3. workflow 
	<mark style="background: #FFF3A3A6;">To meet future requirements</mark> and in <mark style="background: #2dba4e;">making database future ready</mark>

13. Database Administrators (Database admins) are also responsible to keep a check on data integrity, data consistency, data redundancy, hardware and software installation requirements, and routine maintenance of the databases. -They are also responsible for handling data loss, which can be caused due to any error or even due to system failures.
 
### 2 Database Designers
1. <mark style="background: #FFB8EBA6;">Design and create the structure</mark> of the database including triggers, indexes, schemas, entity relationships, tables, constraints, etc.
2. <mark style="background: #FFB86CA6;">Gather database related information</mark> as per requirements like the <mark style="background: #FFF3A3A6;">layout</mark>, <mark style="background: #FFF3A3A6;">looks</mark>, <mark style="background: #FFF3A3A6;">database</mark><mark style="background: #FFF3A3A6;"> functioning</mark>, costing, technologies to be used & implementation techniques.
3. They <mark style="background: #FFB8EBA6;">design the final layout of the database for programmers to code its logic</mark>
4. Responsible for <mark style="background: #FFB86CA6;">implementing the overall design</mark> of the database.
5. They decide which form of data needs to be stored, what kind of relations exist among different entities of the database, what will be the type of attributes etc. 
### 3. Naive Users / Parametric Users
1. Don't have any knowledge of DBMS but still frequently use the database applications to get the desired results.
2. Use the database to fill in or retrieve the information
3. For examples, <mark style="background: #FFF3A3A6;">Railway’s ticket booking users, Clerks in any bank</mark>
4. because they <mark style="background: #2dba4e;">don’t have any DBMS knowledge but they still use the database and perform their given task. </mark>
### 4. System Analysts
1. Analyzes the requirements of parametric end users.
2. check whether all the requirements of end users are satisfied.
3. Responsible for the design, structure & functioning of the database.
4. They usually check and gather all the necessary information related to the database, and if needed, they can change or update the final layout of the database as per requirements
5. System Analysts<mark style="background: #FFF3A3A6;"> always make sure that the final product should meet all the requirements.
</mark>
### 5. Application Programmers / Back-End Developers
1. Back-End Developers, are computer professional users who are responsible for developing the application programs (C, C++, Java, PHP, Python, etc.) or the user interface so that other users can use these applications to interact with the database.
2. Application Programmers have deep knowledge of DBMS & databases and know everything in detail.
3. They interact with the database using DML (Data Manipulation Language) queries to store data inside the database and when needed, they can also fetch the data from it.
4. They are efficient enough in designing or developing their database in any language they know.

### 6. Sophisticated Users
Sophisticated users are the type of database users in ==DBMS who know DBMS and are familiar with the database.== - Sophisticated users can be <mark style="background: #2dba4e;">business analysts, engineers, scientists, system analysts</mark>, etc. - Sophisticated users can develop or access their database applications according to the requirements, - These users are <mark style="background: #FFB86CA6;">also known as SQL programmers</mark> - ==They can interact with the database directly using SQL queries using query processors. - Using SQL queries, they can fetch the data from the database. They can also delete, update or insert new data into the database==.

### 7. Casual Users / Temporary Users
Casual user’s also known as temporary users, are the type of database users in DBMS who frequently or occasionally use the database services. -Whenever these users try to access the database, they want all the information sorted in place. -Casual/Temporary users have ==little knowledge about DBMS and each time they try to access the database, they require new information.== -For example: High-level management people are casual users who have little knowledge about DBMS and hence, they can access the database to either fill in new information or retrieve existing results.
