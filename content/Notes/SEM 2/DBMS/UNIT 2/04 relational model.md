## Relational database mode

- **Edgar Codd 1970**
1. The relational model represents how <mark style="background: #FFF3A3A6;">data is stored in Relational Databases.</mark>
2. A relational database consists of a <mark style="background: #ADCCFFA6;">collection of tables, each of which is assigned a unique name.</mark>
3. A relational database <mark style="background: #FFB8EBA6;">stores data in the form of relations (tables)</mark>.
4. Relational model can represent as a table with<mark style="background: #ADCCFFA6;"> columns and rows. </mark>
5. ROW - **TUPLE**
6. COLUMN - **ATTRIBUTE
7. Column: The column represents the set of values for a specific attribute.
8. Attribute: Each column in a Table. <mark style="background: #FFB86CA6;">Attributes are the properties which define a relation. e.g., Student's Roll </mark>no, NAME, etc.
9. A table has two properties rows and columns. <mark style="background: #FFB86CA6;">Rows represent the records/tuples and columns represent attributes.</mark>
10. Tuple : singe row, single record
11. Relation Schema:<mark style="background: #BBFABBA6;"> A relation schema defines the structure of the relation and it represents the name of the relation with its attributes.</mark>
12. e.g.; student (ROLL_NO, NAME, ADDRESS, PHONE, and AGE) is the relation schema for student.
13. Degree: <mark style="background: #FFB86CA6;">The total number of attributes</mark> which in the relation is called the degree of the relation.
14. Cardinality:<mark style="background: #FFB86CA6;"> Total number of rows </mark>present in the Table.


![](https://i.imgur.com/hTDIyND.png)
14. **<mark style="background: #FFB86CA6;">Relation Instance</mark>**: The set of tuples of a relation at a particular instance of time is called a relation instance. It can change whenever there is an insertion, deletion, or update in the database.
15. <mark style="background: #FFF3A3A6;">**NULL values**</mark>: The value which is not known or unavailable is called a NULL vale. It is represented by blank space.

16. <mark style="background: #FFB8EBA6;">**Relation Key**</mark>: These are basically the keys that are used to identify the rows uniquely or also help in identifying tables. These are of the following types. Primary, foreign, candidate etc.
